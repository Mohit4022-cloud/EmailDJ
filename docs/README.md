# Docs Structure

## Folders
- `architecture/`: runtime topology, request lifecycle, repo inventory
- `contracts/`: API/SSE contract docs and generated OpenAPI summaries
- `policy/`: control invariants and enforcement model
- `ops/`: runbooks, env matrix, DocOps operating model
- `product/`: product intent and positioning guardrails
- `adr/`: architecture decisions
- `_meta/`: glossary + path-to-doc coverage map
