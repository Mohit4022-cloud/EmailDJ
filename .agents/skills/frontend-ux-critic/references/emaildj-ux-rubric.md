# EmailDJ UX Rubric

Use this rubric to rank fixes before any code changes.

## Rank 1: Blocking UX Debt

Raise to the top when the issue:

- hides the main task
- makes the user scan too many controls before acting
- separates status from the active workflow
- makes it unclear where to start
- makes the draft feel less important than the setup form

## Rank 2: High Friction

Raise when the issue:

- causes repeated context switching
- mixes advanced controls with required inputs
- uses weak or low-signal copy
- spreads related fields across disconnected areas
- makes sliders or preset tools compete with generation itself

## Rank 3: Polish Gaps

Raise when the issue:

- weakens perceived quality
- creates minor spacing inconsistency
- leaves empty states or loading states underdesigned
- makes secondary actions look as important as primary ones

## Current Repo Watchlist

- Seller and prospect inputs appear in one long uninterrupted flow.
- Research and enrichment actions are interleaved with core manual input.
- Generate, save remix, and preset browsing compete for the same action row.
- The runtime badge sits above the hero and can read as disconnected system chrome.
- The editor area can lose visual dominance once the input panel grows.

## Review Style

- Be direct.
- Cite the specific area or component causing the issue.
- Prefer a smaller number of high-impact fixes over a long list of cosmetic notes.
